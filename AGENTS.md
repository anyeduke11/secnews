# SecNews Knowledge Dashboard

## Overview
This project contains a knowledge management dashboard (v1.4+).
The `knowledge/` directory is an LLM-Wiki — a filesystem-based
knowledge base readable and writable by both humans and AI agents.

## How to Use This Knowledge Base

### Data Sources
- **Cubox** (via cubox-cli): saved articles with full text + annotations
- **Browser Bookmarks**: imported JSON, deduplicated
- **SecNews Archive**: items favorited/archived from the hotspot dashboard

### Directory Structure
```
knowledge/
├── _MAP.md          ← Knowledge map (auto-generated index)
├── _SCHEMA.md       ← Data model contract
├── SOUL.md          ← Role profile (auto-generated by Agent)
├── items/           ← L1: Knowledge items (*.md with YAML frontmatter)
├── concepts/        ← L2: Extracted concepts
├── learning/        ← L3: Learning plans + progress
│   └── tasks/
│       ├── pending/     ← Agent monitors this directory
│       ├── processing/  ← Agent moves tasks here while working
│       ├── done/        ← Completed tasks
│       └── failed/      ← Failed tasks + error.md
├── content/         ← L4: Content creation
│   ├── drafts/         ← Draft markdown files
│   └── calendar.json   ← Publishing calendar
└── summaries/       ← Weekly digests / reviews
```

### Available Skills
- **cubox**: Card list, RAG search, export
- **knowledge-master**: Learning plan generation, teaching, testing, mastery assessment
- **knowledge-base-manager**: Classification, tagging, knowledge graph, auditing
- **baoyu-post-to-wechat/x/weibo**: Content publishing
- **baoyu-slide-deck/infographic/cover-image**: Content creation

### Task Queue
- Pending tasks: `knowledge/learning/tasks/pending/`
- Task format: Markdown file with YAML frontmatter
- Agent scans pending/ → processes → moves to done/ or failed/

### Federation
This wiki federates with the local LLM-Wiki at `~/knowledge-base/`.
Cross-reference syntax:
- `[[wiki:local:concepts/zero-trust]]` — reference local wiki concept
- `[[wiki:hotspot:items/a1b2c3]]` — reference hotspot wiki item

### Schema
See `knowledge/_SCHEMA.md` for the complete data model.
