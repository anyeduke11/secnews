1. 项目概述

Hotspot 是一个单进程本地热点资讯聚合与质量管控平台。从多个公开数据源(RSS、API、HTML 站点)自动采集热点资讯,经过 10 道质量门禁(9 同步 + 1 异步)打分后入库 SQLite,React 前端展示给用户。

核心能力

多源采集
7 大分类 (AI/安全/金融/创业/招标/GitHub/Tech),103 个源,aiohttp + crawl4ai + JSON + RSS 多渲染器
质量门禁
10 道门禁 (Schema/Content/Category/URL/SourceReputation/TitleSummary/Duplicate/Author/FinalURL/AsyncURL),评分 0-100
趋势分析
按小时聚合 24h 趋势热力图,7 分类 × 24h = 168 桶
定时调度
APScheduler 驱动 5min 全量采集 / 5min 趋势重算 / 5min URL 内容验证 / 6h 源信誉重算
代理感知
off / auto / manual 三种代理模式,支持全局 / 单源覆盖
可观测性
结构化 JSON 日志 + trace_id + 事件打点 + /api/health + /api/stats + source_stats 表

关键非功能指标

指标目标
启动时间< 3s (含冷启动 + 首次采集)
API P95< 200ms (缓存命中 < 50ms)
数据量优雅支撑 1k → 100k+ 条热点
故障恢复服务重启数据零丢失;外部源失败不影响整体可用
进程模型单进程 + 嵌入式 SQLite + 进程内调度,零外部服务依赖

3. 请求生命周期

用户操作 → React 前端 → HTTP API → FastAPI Router
  → TraceIDMiddleware (注入 trace_id)
  → Service Layer (读 list_cache / detail_cache / static_cache)
  → 未命中 → Repository (SQLite + FTS5)
  → 返回响应 + X-Trace-Id / X-Duration-Ms 头

4. 采集生命周期 (5 分钟周期)

APScheduler.collect_all_job
  → CollectionService.run_once()
    → asyncio.gather(7 个 Collector.collect())   并发抓取
      → 每个 Collector.fetch_source() 走 4 种 renderer:
         a. aiohttp      直连 (静态 HTML / RSS / JSON)
         b. crawl4ai     Playwright (反爬 / JS SPA)
         c. rss          feedparser (Phase 22)
         d. json         Phase 25 AIhot / 金十数据
      → _parse_html() + _build_items() → HotspotItem
      → _run_quality_gates() 跑 9 道同步门禁 (thread pool)
    → HotspotRepository.upsert_many()  单事务批量入库
    → TrendRepository.rebuild(24)     重算趋势 168 桶
    → cache_invalidate("hotspots:*") + cache_invalidate("trends:*")
    → 异步触发 url_content_check (抽样 10%)
    → evaluate_source_coverage()  写 source_stats
    → 写 collection_runs  审计日志

7. 核心数据模型

HotspotItem
├── id           str            "{source}_{name}_{i}"
├── title        str            1-500 字符
├── summary      Optional[str]  ≤500 字符
├── source       str            数据源名
├── url          HttpUrl        Pydantic v2
├── category     Category       ai/security/finance/startup/bid/github/tech
├── published_at datetime       tz-aware UTC
├── fetched_at   datetime       tz-aware UTC
├── ingested_at  datetime       入库时间 (排序键, 替代 published_at)
├── score        Optional[int]  0-100
├── quality_score int           0-100
├── quality_flags list[str]     ["title_too_short", ...]
├── bid_status   Optional[str]  Phase 21: 招标中/中标/变更/终止
└── is_fallback  bool = False
