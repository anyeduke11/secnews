# Skills，把方法变能力

用 AI 做事，最容易被忽略的成本，是重复交代。同样是写周报，每次都要说明结构。同样是分析数据，每次都要强调口径。同样是改文档，每次都要提醒风格、标准和禁忌。问题不在于任务有多复杂，而在于方法没有被沉淀下来。Skills 的价值，正是把这些方法整理成一个可识别、可调用、可复用的能力包。一个 Skill 通常从 SKILL.md 开始。它写清楚三件事：适用什么场景，按照什么流程执行，最终结果应该达到什么标准。在此基础上，还可以加入脚本、模板、参考资料和素材。这让 Skills 更像一份“方法说明书”：场景出现时，系统先判断是否相关；相关时读取核心说明；需要更多细节时，再调用模板、规则或脚本。这样...
